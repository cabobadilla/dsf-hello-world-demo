Dark Software Factory

An autonomous multi-agent pipeline that designs, builds, tests, and deploys software — orchestrated by Hermes, executed by specialized AI agents.

What it is

Autonomous Delivery System

A fully autonomous software delivery system using 4+ specialized AI agents. Each agent has a single role, sleeps when idle, and wakes only for its phase.

How it runs

Hermes Orchestrates

Hermes (the orchestrator) receives requests, refines requirements, delegates to agents, judges output, and reports results. No human intervention between phases.

What it costs

~$10 Per Project

Target: ~$10 per project. Haiku 4.5 for mechanical tasks ($1/$5 MTok), Sonnet 4.6 for reasoning ($3/$15 MTok). Budget enforced per agent.

⚠️ Critical Rules

Non-Negotiable System Rules

These rules are non-negotiable. They exist because violations have occurred and been corrected.

Rule 01
Hermes Never Codes

Hermes is Orchestrator + Judge ONLY. If Hermes finds itself writing implementation code → STOP → delegate to Developer Agent. No exceptions.

Rule 02
PO Agent Runs First, Always

Every change, no matter how small, goes through PO Agent first. There are no 'small changes' that skip the process.

Rule 03
Stories Before Code

User stories with acceptance criteria must be created in Plane.so BEFORE any Developer Agent is invoked. Retrospective stories are violations.

Rule 04
Judge Every Output

Hermes reviews ALL agent output before proceeding. Vague language, missing criteria, or scope creep = rejected and re-run.

Rule 05
3 Checkpoints Only

User touches exactly 3 points: (1) confirm brief, (2) approve stories, (3) see live result. Everything else runs autonomously.

Rule 06
TDD Strict

Developer Agent writes failing tests FIRST (RED), then minimal code to pass (GREEN), then refactors. No code before tests.

Rule 07
One Plane Project per DSF Project

No 'DSF —' prefix in project names. Workspace context is sufficient. Each project maps 1:1 to a Plane board.

Rule 08
Budget Enforced Per Agent

Haiku for mechanical tasks, Sonnet for reasoning. Never Opus. Total target: ~$10 per project. Caps enforced before invocation.

Rule 09
Sonnet for Reasoning, Haiku for Structure

Architect and Developer use claude-sonnet-4-6. PO, QA, DevOps, Reviews use claude-haiku-4-5.

Rule 10
Git Reset Before Every Agent

Before any Developer Agent touches the repo: git fetch origin && git reset --hard origin/main. Prevents divergence bugs.
